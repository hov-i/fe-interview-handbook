# 작성 규칙

## 파일명 규칙

- 경로: `weeks/week-XX/{본인 GitHub 핸들}.md`
- 예시: `weeks/week-01/hovi.md`
- 주차 번호는 두 자리로 패딩 (week-01, week-02, ...)

## 질문 파일 구조

각 질문은 `## ` (h2)로 시작하며, 바로 아래에 카테고리를 작성합니다.
질문 사이는 `---` 구분자로 구분합니다.

```markdown
## 질문 제목
- **카테고리**: JavaScript

### 답변
답변 내용

### 꼬리질문
- 꼬리질문 1
- 꼬리질문 2

---

## 다음 질문...
```

## 허용 카테고리

다음 값만 사용할 수 있습니다:

| 카테고리 |
|----------|
| JavaScript |
| TypeScript |
| React |
| CSS |
| HTML |
| Network |
| Browser |
| CS |
| 기타 |

## 최소 질문 수

한 파일에 **최소 10개** 이상의 질문을 작성해야 합니다.

## PR 규칙

### 제목 컨벤션

```
[week-XX] {핸들} 질문 N개
```

예시: `[week-01] hovi 질문 10개`

### PR 프로세스

1. `main` 브랜치에서 새 브랜치를 생성합니다.
2. `TEMPLATE.md`를 복사하여 `weeks/week-XX/{핸들}.md`로 저장합니다.
3. 질문/답변/꼬리질문을 작성합니다.
4. PR을 올리면 자동으로 메타데이터 검증이 실행됩니다.
5. 검증 통과 + 리뷰 완료 후 머지합니다.

## 머지 후 자동 갱신

PR이 `main`에 머지되면 GitHub Actions가 자동으로 실행되어:

1. `weeks/` 폴더의 모든 마크다운 파일을 스캔합니다.
2. 각 질문의 카테고리를 파싱합니다.
3. `README.md`의 질문 인덱스를 카테고리별로 갱신합니다.
4. 같은 제목의 질문은 하나로 묶어 각 작성자 링크를 표시합니다.
