# 📝 작성 가이드

프론트엔드 기술면접 스터디에 오신 걸 환영합니다!
아래 규칙을 따라 질문 파일을 작성해주세요.

---

## 📁 파일명 규칙

```
weeks/week-XX/{본인 GitHub 핸들}.md
```

- 주차 번호는 **두 자리**로 작성 (week-01, week-02, ...)
- 파일명은 본인 GitHub 핸들과 동일하게
- 예시: `weeks/week-01/hovi.md`

---

## ✍️ 질문 작성 포맷

각 질문은 `## `(h2)로 시작하고, 바로 아래에 카테고리를 작성합니다.
질문 사이는 `---`로 구분합니다.

```markdown
## 이벤트 루프가 어떻게 동작하나요?
- **카테고리**: JavaScript

### 답변
콜 스택, 태스크 큐, 마이크로태스크 큐로 구성되어 있고...

### 꼬리질문
- 마이크로태스크와 매크로태스크의 차이는?
- requestAnimationFrame은 어디에 속하나요?

---

## 다음 질문 제목
- **카테고리**: React

### 답변
...
```

---

## 📂 허용 카테고리

아래 값만 사용할 수 있습니다. 이외의 값은 **PR 검증에서 실패**합니다.

| 카테고리 | 설명 |
|----------|------|
| `JavaScript` | JS 핵심 개념, ES6+, 비동기 등 |
| `TypeScript` | 타입 시스템, 제네릭, 유틸리티 타입 등 |
| `React` | 컴포넌트, 훅, 상태관리, 렌더링 등 |
| `CSS` | 레이아웃, 애니메이션, 방법론 등 |
| `HTML` | 시맨틱, 접근성, SEO 등 |
| `Network` | HTTP, 캐싱, 보안, REST 등 |
| `Browser` | DOM, 렌더링 파이프라인, 이벤트 등 |
| `CS` | OS, DB, 네트워크 기초 등 |
| `자료구조` | 리스트, 트리, 해시 등 |
| `빌드도구` | Webpack, Vite, CI/CD 등 |
| `기타` | 위에 해당하지 않는 주제 |

---

## 🔢 최소 질문 수

한 파일에 **최소 10개** 이상의 질문을 작성해야 합니다.

---

## 🚀 PR 규칙

### 브랜치 & 제목

```
브랜치: week-01/hovi
PR 제목: [week-01] hovi 질문 10개
```

### 프로세스

1. `main`에서 새 브랜치를 생성합니다.
2. `TEMPLATE.md`를 복사해서 `weeks/week-XX/{핸들}.md`로 저장합니다.
3. 질문 / 답변 / 꼬리질문을 작성합니다.
4. PR을 올리면 **자동으로 검증**이 실행됩니다.
   - 카테고리가 허용 목록에 있는지
   - 질문이 10개 이상인지
5. 검증 통과 + 리뷰 완료 후 머지합니다.

---

## 🔄 머지 후 자동 갱신

PR이 `main`에 머지되면 GitHub Actions가 자동으로:

1. `weeks/` 폴더의 모든 질문 파일을 스캔
2. 카테고리별로 질문을 분류
3. 같은 제목의 질문은 하나로 묶어 각 작성자 링크 표시
4. `README.md`의 질문 인덱스를 갱신

> 별도 작업 없이 머지만 하면 README가 알아서 업데이트됩니다!
