## www.google.com을 접속할 때 웹 브라우저 요청 흐름을 패킷 관점에서 설명해주세요.

- **카테고리**: Network

### 답변

사용자가 www.google.com을 입력하면 먼저 DNS를 통해 도메인 이름을 IP 주소로 변환합니다.
이후 목적지와 통신하기 위해 같은 네트워크라면 목적지의 MAC 주소를, 다른 네트워크라면 기본 게이트웨이의 MAC 주소를 ARP를 통해 조회합니다.

그다음 TCP의 3-way Handshake로 연결을 수립한 뒤 HTTP 요청을 전송합니다.
이 요청은 애플리케이션 계층에서 생성되어 TCP 세그먼트, IP 패킷, 이더넷 프레임으로 캡슐화되어 네트워크를 통해 전달됩니다.
서버는 요청을 처리한 뒤 응답을 보내고, 브라우저는 받은 HTML을 파싱하여 화면을 렌더링합니다.

- **카테고리**: Network

### 답변

TCP는 전송 계층에서 사용하는 연결 지향 프로토콜입니다.
데이터를 보내기 전에 3-way Handshake로 연결을 맺고, 시퀀스 번호, ACK, 재전송 등의 기능으로 데이터의 순서와 신뢰성을 보장합니다.
장점은 데이터 손실 없이 순서대로 전달된다는 점이며, 단점은 연결 설정과 유지 비용 때문에 UDP보다 속도가 느리다는 점입니다.

## HTTP의 비연결성과 무상태성을 설명해주세요.

- **카테고리**: Network

### 답변

HTTP는 기본적으로 비연결성과 무상태성을 가집니다.
비연결성은 요청과 응답이 끝나면 연결을 종료하여 서버 자원을 효율적으로 사용하는 특징입니다.
무상태성은 서버가 이전 요청을 기억하지 않는 것으로, 필요한 정보는 매 요청마다 함께 전달해야 합니다.
로그인과 같이 상태 유지가 필요한 기능은 쿠키, 세션, JWT 등의 기술을 이용해 구현합니다.

## HTTP 1.0 / 1.1 / 2.0 / 3.0의 차이점을 설명해주세요.

- **카테고리**: Network

### 답변

HTTP/1.0은 요청마다 새로운 TCP 연결을 생성했습니다.
HTTP/1.1은 Keep-Alive를 통해 연결을 재사용하여 성능을 개선했습니다.
HTTP/2는 바이너리 프레이밍과 멀티플렉싱을 도입하여 하나의 연결에서 여러 요청을 동시에 처리할 수 있게 했고, 헤더 압축도 지원합니다.
HTTP/3는 TCP 대신 QUIC(UDP 기반)을 사용하여 연결 시간을 줄이고 Head-of-Line Blocking 문제를 개선했습니다.

## GET과 POST의 차이점은 무엇인가요? PUT과 PATCH는 어떻게 다른가요?

- **카테고리**: HTTP

### 답변

GET은 리소스를 조회하는 메서드이며 서버 상태를 변경하지 않는 안전하고 멱등한 메서드입니다.
POST는 새로운 리소스를 생성하거나 특정 작업을 수행할 때 사용하며 일반적으로 멱등하지 않습니다.
PUT은 리소스를 전체 교체하는 의미이며 같은 요청을 반복해도 결과가 같기 때문에 멱등합니다.
PATCH는 리소스의 일부만 수정하는 메서드이며 구현 방식에 따라 멱등성이 달라질 수 있습니다.

- GET 요청에 Body를 넣어도 될까요?

## HTTP 메서드의 멱등성이 무엇이고 왜 중요한가요?

## NAT(Network Address Translation)란 무엇인가요?

- **카테고리**: Network

### 답변

NAT(Network Address Translation)는 사설 IP 주소를 공인 IP 주소로 변환하거나, 반대로 공인 IP를 사설 IP로 변환하는 기술입니다.
일반적으로 가정이나 회사에서는 여러 기기가 사설 IP를 사용하고, 공유기(Router)가 NAT를 통해 하나의 공인 IP로 인터넷과 통신합니다.
이를 통해 공인 IP 주소를 효율적으로 사용할 수 있으며, 내부 네트워크 구조를 외부에 직접 노출하지 않아 보안 측면에서도 도움이 됩니다.
다만 NAT를 사용하는 경우 주소 변환 과정이 필요하기 때문에, 외부에서 내부 장치로 직접 접근하려면 포트 포워딩과 같은 추가 설정이 필요할 수 있습니다.

- 공인 IP와 사설 IP의 차이는 무엇인가요?
- 포트 포워딩이 필요한 이유는 무엇인가요?
