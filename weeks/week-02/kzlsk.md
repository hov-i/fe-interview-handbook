# N주차 - 이름

## 호이스팅이란?
- **카테고리**: JavaScript

### 답변
- 변수와 함수의 선언문이 해당 스코프의 최상단으로 끌어올려지는 현상
- 초기화되기 전에 변수와 함수에 접근 가능
- But, let과 const는 TDZ로 인해 접근 시 에러 발생

---

## 클로저란?
- **카테고리**: JavaScript

### 답변
- 클로저는 함수와 그 함수가 선언될 당시의 렉시컬 환경의 조합
- 내부 함수가 외부 함수 참조 시 외부 함수의 실행이 끝나도 그 변수가 사라지지 않고 계속 남아있기 때문에 계속 참조 가능

---

## 스코프란?
- **카테고리**: JavaScript

### 답변
- 식별자가 유효한 범위
- 자바스크립트 엔진은 스코프를 통해 어떤 변수를 참조해야 할 것인지를 결정하기 때문에 식별자를 검색할 때 사용하는 규칙이라고도 할 수 있음

---

## this란?
- **카테고리**: JavaScript

### 답변
- 자신이 속한 객체 또는 자신이 생성할 인스턴스를 가리키는 자기 참조 변수
- this는 자바스크립트 엔진에 의해 암묵적으로 생성되며, 코드 어디서든 참조 가능
- this를 통해 자신이 속한 객체 또는 자신이 생성할 인스턴스의 프로퍼티나 메서드 참조 가능
- this가 가리키는 값, 즉 this 바인딩은 함수 호출 방식에 의해 동적으로 결정

---

## 화살표 함수와 일반 함수의 차이는?
- **카테고리**: JavaScript

### 답변
- 화살표 함수의 this는 언제나 상위 스코프의 this를 가리키며 arguments 객체가 없고, new 생성자로 호출
- 일반 함수는 호출 방식에 따라 this가 동적으로 결정

---

- **카테고리**: JavaScript

### 답변
- 소스코드를 실행하는 데 필요한 환경을 제공하고 코드의 실행 결과를 실제로 관리하는 영역
- 실행 컨텍스트 스택과 렉시컬 환경으로 구성

---

## 동기와 비동기의 차이는?
- **카테고리**: JavaScript

### 답변
- 동기 처리 방식은 현재 실행 중인 작업이 종료될 때까지 기다렸다가 다음 작업이 실행되는 방식
- 비동기 처리 방식은 현재 실행 중인 작업이 종료되지 않은 상태여도 다음 작업을 곧바로 실행하는 방식

---

---

## Virtual DOM이란?
- **카테고리**: React

### 답변
- 실제 DOM의 가벼운 복사본을 메모리에 유지하는 개념
- 상태 변경 시 새로운 Virtual DOM 트리를 생성하고, 이전 트리와 비교(diffing)하여 변경된 부분만 실제 DOM에 반영(reconciliiation)

## Flex와 Grid를 사용하는 이유는?
- **카테고리**: CSS

### 답변
- flex와 grid는 레이아웃을 쉽게 잡기 위한 속성
- flex
    - 1차원 레이아웃에 최적화된 속성
    - 컨테이너와 아이템 개념을 사용해 한 방향으로 요소를 정렬하고, 요소의 크기가 불분명하거나 동적인 경우에도 효율적으로 배치 가능
- grid
    - 2차원 레이아웃에 최적화된 속성
    - 간격과 비율만 저장하면 행과 열 동시에 제어 가능하기 때문에 복잡한 레이아웃을 간단하게 구성 가능
    - 브라우저 창 크기가 변해도 자동으로 반응하기 때문에 반응형 레이아웃을 만들기 쉬움

## CORS란?
- **카테고리**: CS

### 답변
- 교차 출처 공유
- 브라우저가 다른 출처로의 요청을 허용할지 판단하는 보안 메커니즘
- 브라우저는 기본적으로 동일 출처 정책에 따라 다른 출처로의 요청을 차단하는데, 서버가 응답 헤더에 허용 출처를 명시하면 브라우저가 이를 확인하고 응답을 허용

## 브라우저가 HTML을 받아서 화면에 그리기까지의 과정을 설명해주세요.
- **카테고리**: CS

### 답변
- HTML 파싱 > DOM 트리 생성 > CSS 파싱 > CSSOM 생성 > Render Tree 결합 > Layout(reflow) > paint > 레이어 합성(composite)
- HTML 파싱해서 DOM 트리 생성하고, CSS 파싱해서 CSSOM 생성 후 둘을 합쳐 Render Tree 구성
- 이후 각 요소의 크기와 위치 계산하는 Layout, 화면에 색을 채우는 paint, 레이어 합성 순서로 진행

---

- **카테고리**: Network
