# 프론트엔드 기술면접 스터디

## 소개

> 신입/주니어 프론트엔드 개발자를 위한 기술면접 대비 스터디입니다.
> 목적, 기간 등은 추후 업데이트 예정입니다.

## 진행 방식

1. 매주 `weeks/week-XX/{본인 핸들}.md` 파일을 만들어 PR을 올립니다.
2. 한 파일에 최소 **10개** 질문 + 답변 + 꼬리질문을 작성합니다.
3. 스터디 시간에 서로 질문하고 리뷰 후 PR을 머지합니다.
4. 머지되면 GitHub Actions가 자동으로 아래 질문 인덱스를 갱신합니다.

## 폴더 구조

```
.
├── .github/workflows/   # CI/CD
├── scripts/             # 자동화 스크립트
├── weeks/               # 주차별 질문 파일
├── TEMPLATE.md          # 질문 파일 템플릿
├── CONTRIBUTING.md      # 작성 규칙
└── README.md
```
